Internship Projects

  • Speech to Text Engine - Apache Stanbol

    An OSGi module built in Maven framework for processing audio data.

    Mentor: Andreas Kuckartz

    Abstract:May - August, 2014

    The proposed project is related to the issue STANBOL- 1007 for the development of enhancement engine for open source project Apache Stanbol.

    Stanbol is a web tool that provides reusable components for semantic content management. It provides a Semantic services for Content Management System via RESTful api.

    Developed enhancement engine converts speech to text. It uses CMU Sphinix library to convert the encoded audio in a media file. ContentItem object provided by Stanbol parses the media file.

    Currently media data is formatted to compatible format using FFMPEG libraries. Text is extracted from the audio and Semantically Enhanced keeping track of the temporal position of the extracted text. Sphinix uses acoustic model and language model to map the utterances with the text, so the engine will also provide facility of uploading acoustic model and language model using DataFileProvider service.

    The GSOC project will allow the development of Phonetic based EntityLinking(STANBOL- 1291) engine for the Stanbol.

  • Probabilistic Representation and Recognition of Historical Text - CVIT

    Optimizing the traditional recognition technique of Scene Text Recognition.

    Mentor: Dr. C. V. Jawahar

    Abstract: May - June, 2013

    Project implements the idea of optimizing the traditional recognition techniques used in Scene Text Recognition. The process is implemented using OpenCV libraries

    Training set used comprised of english word images. Characters are extracted from them abd feature vector is generated on the basis of the paper [3].

    The feature vector of character images is trained by SVM using multiclass classification. SVM link provided in the text were used for multiclass classification as OpenCV do not had that service. Normailsed training values were grouped to form training set of word images.

    Above process achieved efficiency of 77% and is less than other studies.

    Further approach invloved to generation of a candidate window of word images. Get their corresponding word tag and improve the results based on its ngram model. .

    Result Set:

    CVIT word image data set were used for prediction. This project was dropped as the results were not efficient enough to continue.

AVATAR - Assistive Technology for Underprivileged people

Suman Saurabh, Gajendar Singh Tanwar
Independent Project Assistive Technology

Abstract:

Engineered a device Avatar - goggle that helps hand disabled people to interact with their computer. With goggle resides an infra-red light which is tracked by engineered camera. Software takes Camera feed is taken and send it driver that stimulates the feed on mouse cursor. Software and driver uses TUIO Protocol to communicate.

Left Click and Right Click happens by by another infra-red light which glows with peaks in the voice.

Basic Voice Command recognition facility has also been added to it, it uses pockephoenix api’s for command recognition. Command can be used to perform left and right click - single as well as double, opening browser and music players.

Practical Application: Devised prototype costs approximately 12$ and is significantly cheaper and better than their counterparts hence can be used alternative to the devices like using brainwaves, eye tracking, e.t.c

Knowledge Base Compiler

Suman Saurabh, Utkarsh Jain, Ashish Soni and Shirish Singh
Academic Project: Artificial Intelligence

Abstract:

Developed Rule Based Engine in JAVA that performs simple lexical analysis on rules, parses them for facts and perform deduction using Forward or Backward Chaining inference engine.

Forward Chaining Inference Engine evaluates the parsed facts and derive the goal from the rules present in database. Backward Chaining inference Engine is used to detect the validity of goal by asking about the validity of its corresponding basic facts.

Output for the inferences drawn are displayed in graphical tree format.

AngelApi Job extractor

Suman Saurabh
Independent Project Google App Engine, Python

Abstract:

This web application is a quick hack developed to get list of particular type of jobs using AngelList api‘s. The AngelList Bot basically gets all startups given an input city (e.g. NYC) and keyword, than checks AngelList jobs available.

Google NoSQL database is used to hold the information of Keyword, Location tags to the coresponding Job id's. For a query a list of job ids are retrieved and information is fetched from AngelList Sever.

AngelList api's lags to fetch results in real-time, so various google app engine services are used like memcache, to meet the real time needs.
