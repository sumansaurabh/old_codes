Serving static website

When you do not have a credit card

Posted by Suman Saurabh on October 26, 2014

Static sites are the most basic type of website and are the easiest to create. Unlike dynamic websites, they do not require any Web programming or database design. They are mostly used for blogging or creating portoflios.

You’ve put in all the hard work of creating a static website, blog, you want to get it on the web so people can access its content. But you might be a student, Indian student, or any person who don't have an access to a credit card. I was in a similar situation and I found Google App Engine(GAE) doesn't need credit card to deploy a static website.

In this blog we will be covering:

  1. Deploy and Host a website We will get a domain from GAE, similar to http://game-2048.appspot.com We will be deploying the Game-2048 from https://github.com/gabrielecirulli/2048

  2. Creating your own mail-server Similar to info@game-2048.appspot.com

  3. Hosting to a free .tk domain Yup it will be free.

To deploy, you should have a bit of python knowledge, if you need to dirty your hands.

Your website must have been ready by now. In case you are a newbie, please check out the tons of resource avaialable on Internet

1. Installing Google App Engine

Since we will be using GAE-Python to host, here is the link to install GAE in your machine: GAE Installation. Please follow the steps correctly as documentation is bit unclear.

2. Creating a GAE-Project

To use Google's tools for your own site or app, you need to create a new project on Google Cloud Platform from your Google account. Go to the App Engine dashboard on the Google Cloud Platform Console and create a project.

When creating App engine application from, the application id is very important. Because it acts as subdomain for your site. Lets say, application id is game-2048, the site will be in game-2048.appspot.com.

3. Loading source code to GAE

  1. a.


    We will need a sample GAE app or download it from the link.

  2. b.

    Directory strucutre of the downloaded app is: 

    
├── app.yaml                                  #Holds the configuration file for project
├── config.py                                  #User config file, will be discussed later

    ├── js
    │   ├── *.js
    ├── others
    │   └── *.*

  3. c.

    The A sample content is present in static folder. If the content of your website follows a different strcture, you need to learn to configure app.yaml, which is pretty easy.

    You might be using package manager like bower, npm and it's possible to use those in GAE which is not covered in this blog, but you can use this link to read about it and configure your app. 

    
application: games-2048                          #Will have the name of your GAE application, could be anything like 'gondor'
version: 1                         
runtime: python27
api_version: 1
threadsafe: true


#If you have a different directory structure, you need to edit these handlers.
#The folder '/css' will serve all '*.css' file. If your '*.css' is in '/styles' folder, please add '/styles' in the 'handlers'.


########################### Example #################################### 
# handler:
# - url: /styles
# - static_dir: static/styles
########################################################################

handlers:
- url: /css
  static_dir: static/css

- url: /js
  static_dir: static/js

- url: /images
  static_dir: static/images

- url: /others
  static_dir: static/others


#This is used to build GAE mail server, it's covered later in the blog.
- url: /_ah/mail/info@.*game-2048\.appspotmail\.com
  script: mailredirector.application
  login: admin



- url: /.*
  script: main.application


inbound_services:
- mail

libraries:
- name: webapp2
  version: latest
- name: jinja2
  version: latest

  4. d.

    Adding Mail server to your application

    It will allow you to create your own mailing address like: connect@gondor.appspotmail.com. Any mail sent to this address will be reverted to your mail box. To achieve this, you have to modify config.py and app.yaml file. 

    

####################################### config.py ############################################

#Replace your own mail address like - 'connect@gondor.appspotmail.com'
WEBSITE_MAIL_SERVER="info@game-2048.appspotmail.com"

#Replace with your personal mail id
OWNER_MAIL_ID="johndoe@gmail.com"




              
    

############################# app.yaml #################################

#The above and below code of app.yaml remains the same

#modify 'info@.*game-2048\.appspotmail\.com' to 'connect@.*gondor\.appspotmail\.com'
- url: /_ah/mail/info@.*game-2048\.appspotmail\.com
  script: mailredirector.application
  login: admin

4. Deploying the code


  1. a.

    Testing local deployment

    Run the dev_appserver.py command as follows from the directory that contains your app's app.yaml configuration file: Specify the directory path to your app, for example:

    dev_appserver.py [PATH_TO_YOUR_APP]

  2. b.

    Sending it to Web

    From the same directory run the following command

    gcloud app deploy

    Now navigate your browser to [your-project-id].appspot.com to see your website online. In this case http://game-2048.appspot.com

5. Free [].tk domain


  1. a.

    SignUp

    SignUp on website http://www.dot.tk and check availability of your domain, click on Checkout.

  2. b.

    Configure DNS

    There is no a DNS configuration required. Freenom(.tk provider) facilitates service to redirect to your website url i.e http://game-2048.appspot.com

    Click on Continue and your website will be ready with .tk domain.
